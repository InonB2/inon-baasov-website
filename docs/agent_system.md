# Calvin's Agent System — Inon Baasov Career Architecture

## Architecture Philosophy
Inspired by myICOR.com's team model: every specialist has a SINGLE domain and goes deep. No generalists. No overlap. All agents route through Calvin (orchestrator). Each agent has a self-improvement loop — they review their own output, learn from feedback, and update their own rules file after every major task.

---

## THE ORCHESTRATOR
**Calvin** — Executive Career Architect  
Routes every task to the right specialist. Manages memory. Makes sure nothing falls through the cracks. Knows enough about everything to ask the right question but delegates execution to specialists.

## SELF-IMPROVEMENT LOOP (Universal — All Agents)

Every agent runs this loop after every major task:

```
1. EXECUTE — Complete the assigned task
2. REVIEW — Check output against domain standards
3. LOG — Record: What worked? What didn't? What would I do differently?
4. UPDATE — Modify own rules/frameworks file with the learning
5. REPORT — Send summary to Calvin (orchestrator)
```

## PRODUCT PORTFOLIO FRAMEWORK (Nancy Li + Best Practices)

Synthesized from Dr. Nancy Li (PM Accelerator), myICOR model, and Awwwards winners:

### What Every Product Case Study MUST Include:
1. **Problem** — What user/business pain was being solved? (Specific, quantified)
2. **Process** — How did YOU approach it? (Research, discovery, frameworks used)
3. **Solution** — What did you build/ship? (Concrete, visual if possible)
4. **Your Role** — What specifically did YOU do? (Not the team — you)
5. **Impact** — What changed? (Metrics: revenue, retention, efficiency, scale)
6. **Learnings** — What would you do differently? (Shows maturity)

### Portfolio Conversion Principles (Nancy Li):
- Show OUTCOMES not features — hiring managers care about impact
- Demonstrate END-TO-END thinking — discovery → delivery → iteration
- Include FAILURES + learnings — shows intellectual honesty
- Make it SCANNABLE — 6-second rule for busy hiring managers
- QUANTIFY everything — no metric = no credibility

### Visual Best Practices (Awwwards):
- Dark minimal design converts better for senior/tech roles
- Micro-interactions signal attention to detail
- Typography hierarchy = cognitive ease
- Mobile-first (50%+ recruiters view on mobile)
- Fast load time (< 2s) = professionalism signal
